# Product Requirements Document (PRD)

## Apna Browser Hackathon (6-Hour Build)

---

# 1. Overview

Apna Browser is a lightweight experimental desktop browser that students will build during a short hackathon.

The primary goal is to help students understand the **fundamentals of how browsers work**, while also learning how **AI coding agents can accelerate software development**.

Participants will build a **minimal desktop browser application** capable of:

* Loading websites
* Navigating between pages
* Providing useful productivity tools for students

The project is intentionally designed to be **simple, experimental, and educational**, rather than a full production browser.

The hackathon emphasizes:

* Rapid prototyping
* Creative experimentation
* AI-assisted development

The entire prototype must be built within a **6-hour development window**.

---

# 2. Hackathon Objectives

## Learning Goals

Participants should gain practical experience with:

1. Understanding how browsers display web pages
2. Understanding the structure of browser user interfaces
3. Building desktop applications capable of rendering websites
4. Using AI coding agents to assist in software development
5. Rapid prototyping within tight time constraints

---

## Success Criteria

A participant succeeds in the hackathon if they:

* Build a working browser application
* Successfully load and render websites
* Implement at least **one useful custom feature**
* Demonstrate the browser in a **2–3 minute demo**

---

# 3. Target Participants

## Primary Participants

The hackathon is designed for:

* Students
* Beginner developers
* Participants from Apni Pathshala learning programs

---

## Expected Skill Level

Participants are expected to have:

* Basic understanding of JavaScript or web development
* Ability to use AI coding tools (ChatGPT, Copilot, Codex, etc.)
* Basic familiarity with Git and GitHub

Advanced programming experience is **not required**.

---

# 4. Product Vision

The vision of Apna Browser is:

> "A lightweight experimental browser built by students with AI."

The browser should demonstrate how **students can design their own browsing tools** for productivity, learning, and research.

The project should encourage students to think about:

* What tools are missing from current browsers
* How AI can improve browsing
* How developers can customize the web experience

The browser is **not intended to compete with Chrome or Firefox**.

Instead, it should serve as a **learning platform and experimentation environment**.

---

# 5. Hackathon Deliverables

Each participant or team must submit the following:

## Required Deliverables

1. A working browser prototype
2. Source code uploaded to GitHub
3. A short demo presentation (2–3 minutes)

---

# 6. Technical Approach

Since the hackathon duration is only **6 hours**, participants should rely on frameworks that simplify desktop application development.

Recommended approaches include:

* Tauri
* Electron
* Lightweight desktop WebView wrappers

The browser should rely on an **existing system WebView or browser engine** rather than attempting to implement a rendering engine from scratch.

---

## Required Application Components

Every browser prototype must contain the following components:

### 1. Browser Toolbar

The browser must include a navigation toolbar containing:

* URL input bar
* Back button
* Forward button
* Refresh button

---

### 2. Webpage Rendering Area

The main section of the application must display websites using a **WebView**.

The WebView should be capable of loading external websites via URLs.

---

### 3. Custom Homepage

When the browser launches, it should open a **custom homepage**.

The homepage should include:

* A search box
* Quick links to useful websites
* Basic UI elements designed for students

Example quick links:

* Wikipedia
* YouTube
* Khan Academy
* Educational platforms

---

## AI Assistance

Participants are encouraged to use AI tools to:

* Generate boilerplate code
* Debug issues
* Design UI components
* Implement new features
* Optimize development speed

---

# 7. Mandatory Features

Each browser prototype must include the following features.

---

## 7.1 Basic Browser Interface

The browser must implement:

* URL input bar
* Back button
* Forward button
* Refresh button
* Webpage display area

These features ensure the browser can perform basic navigation tasks.

---

## 7.2 Custom Homepage

The browser should open to a homepage that includes:

* Search box
* Quick access links
* Student-friendly UI

This homepage acts as the **default start page** for the browser.

---

## 7.3 One Custom Feature

Each team must implement **at least one original feature**.

This feature should provide additional functionality beyond the basic browser interface.

Examples include:

* Website blocker
* Reading mode
* Focus timer
* Notes panel
* Screenshot tool

---

# 8. AI-Assisted Feature Ideas

Participants are encouraged to experiment with AI-powered features.

Possible examples include:

---

## Page Summarizer

Summarize the content of the current webpage using an AI model.

Useful for quickly understanding long articles.

---

## AI Study Assistant

Allow users to ask questions about the current webpage and receive AI-generated answers.

---

## Smart Bookmarking

Automatically categorize saved pages using AI.

---

## AI Research Mode

Extract important information and key points from long articles.

---

## Auto Notes

Automatically generate notes from webpage content.

---

# 9. Judging Criteria

Each project will be evaluated according to the following criteria.

| Criteria             | Weight | Description                                    |
| -------------------- | ------ | ---------------------------------------------- |
| Innovation           | 25%    | Creativity and originality of the feature      |
| Practical Usefulness | 25%    | Whether the feature helps users while browsing |
| Technical Execution  | 25%    | Reliability and quality of implementation      |
| Effective Use of AI  | 15%    | How well AI tools were used in development     |
| Demo & Storytelling  | 10%    | Quality of the final presentation              |

---

## Bonus Points

Judges may award additional points for:

* A polished and intuitive user interface
* Multiple working features
* A feature that clearly solves a real student problem
* Creative integration of AI capabilities

---

---

## Hours 2–5

* Rapid development

# 11. Recommended Development Strategy

Because the hackathon is time-constrained, participants should follow this strategy:

1. Build the browser UI first
2. Ensure websites load correctly
3. Implement navigation controls
4. Add a custom homepage
5. Implement one standout feature

AI coding tools should be used to **accelerate implementation and debugging**.

---

# 12. Example Winning Project

### Apna Study Browser

Example concept for a successful project.

Features:

* Minimal browser UI
* AI page summarizer
* Notes sidebar for students
* Quick links to educational resources

This project focuses on **helping students learn faster while browsing**.

---

# 13. Goal of the Hackathon

The primary goal of the hackathon is **not to build a production browser**.

Instead, it aims to:

* Teach browser concepts
* Introduce AI-assisted development
* Encourage rapid experimentation
* Inspire students to build useful tools

The best prototypes may later evolve into **larger open-source projects**.
